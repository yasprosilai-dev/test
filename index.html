О проекте

Что это такое

StrigaPrint — мастерская фотокартин на дереве. Весь процесс от выбора размера до передачи готовой работы автоматизирован через Telegram.

Клиент проходит опрос и сразу видит итоговую стоимость. Администратор ведёт заказ через кнопки прямо в группе — без CRM, без переключений, без потерянных сообщений.

Что заменяет
✗ Ручную переписку в личке
✗ Ручной расчёт стоимости
✗ Забытые заказы и потерю статусов
✗ Изменение прайса через код и деплой
Путь клиента

8 шагов до готового заказа

1
Старт
Клиент пишет /start — получает видео-приветствие и прайс с выравниванием по столбцам. Цены динамические, из базы данных.
2
Выбор размера
8 вариантов от XS 25×30 до XL 75×94. Каждый размер — отдельная кнопка с кодом и ценой.
3
Дополнительный дизайн
Нужно ли объединить несколько фото в одно — да или нет. Добавляет фиксированную сумму к итогу.
4
Фото файлом
Клиент отправляет фото как файл, а не как изображение. Бот ждёт именно документ — для сохранения качества без сжатия Telegram.
5
Упаковка
Три варианта: без упаковки, бежевый крафт или чёрный крафт. Цена упаковки добавляется к итогу.
6
Срок
Клиент вводит дату, к которой нужна картина. Свободный ввод текстом.
7
Доставка
ЕвроПочта (клиент вводит адрес) или самовывоз из точки (выбор из кнопок). Цена зависит от варианта.
8
Подтверждение
Итоговая карточка с полной разбивкой цены. Кнопки: «Отправить заявку», «Изменить доставку», «Заново».
Панель администратора

Управление через группу

Каждый заказ — отдельный топик в Telegram-группе. При поступлении заявки создаётся тред с карточкой заказа и кнопками управления.

3️⃣ Оплата получена
4️⃣ Готово
5️⃣ Передан
✏️ Уточнение
❌ Отменить

После статуса «Готова» открывается двусторонний чат: администратор пишет в топик → клиент получает в боте, и наоборот.

После «Передан» иконка топика меняется на ✅ — визуальный индикатор завершённого заказа.

Ключевые возможности

Что умеет бот

💬

Двусторонний чат

После кнопки «Готово» открывается прямая переписка. Клиент пишет в бот — сообщение появляется в топике. Администратор отвечает из топика — приходит клиенту.

💰

Управление ценами без деплоя

Прайс хранится в PostgreSQL и меняется командами прямо в Telegram. Изменения мгновенно отражаются в приветственном сообщении для новых клиентов.

🧾

Снимок цены на момент заказа

Каждый заказ сохраняет полный snapshot цен: базовая, крафт, упаковка, почта, дизайн. Изменение прайса не влияет на уже созданные заказы.

🔒

Защита от дублей

Пока у клиента есть активный заказ, кнопка /start неактивна. Новый заказ можно оформить только после завершения или отмены текущего.

📋

История событий

Каждое действие пишется в таблицу order_events: кто, что и когда сделал. Полная цепочка от создания заказа до передачи клиенту.

🚀

Мгновенный деплой

git push → автоматический Docker build → бот перезапущен на хостинге. Весь цикл занимает около 40 секунд.

База данных

4 таблицы в PostgreSQL

Нeon Cloud — бессерверный PostgreSQL. Данные переживают перезапуск контейнера, масштабирование и деплои.

orders

Заказы с полным снимком цены на момент оформления. Хранит статус, topic_id, фото, адрес доставки, дату и признак получения чека.

price_items

Актуальный прайс — 8 размеров. Для каждого хранится базовая цена, крафт, упаковка для пересылки и ЕвроПочта.

settings

Глобальные настройки: валюта (BYN) и цена дополнительного дизайна. Меняются без перезапуска бота.

order_events

Лог каждого действия по заказу: тип события, actor_id и заметка. Полная хронология от создания до закрытия.

Деплой

Запуск за 40 секунд

Бот работает в Docker-контейнере на JustRunMy.App. Переменные окружения задаются в панели хостинга — в git не попадают никогда.

# Обычный push на GitHub git push origin main # Деплой на JustRunMy.App одной командой git push https://user:token@justrunmy.app/git/r_Ks2t4HM HEAD:deploy # JustRunMy.App: Docker build → контейнер перезапущен → бот работает

Переменные окружения:

BOT_TOKEN DATABASE_URL ADMIN_IDS ADMIN_GROUP_ID PAYMENT_DETAILS WELCOME_VIDEO PRICE_TABLE_PHOTO PRICE_DESIGN CURRENCY
